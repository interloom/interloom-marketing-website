<h2>Introduction</h2><p>As we have shown in our blog post on <a href="https://interloom.com/en/blog/AI-Native-Organizations_2vDXGLJywCiXbbyxxKOvFpLAtWB" target="_blank"><strong>AI-native organizations</strong></a>, real organizational efficiency gains come not just from using AI agents, but from ensuring the right actors — human or AI agent — are assigned to the right tasks at all times.</p><p>With specialization advantages between agents and humans reaching 100x, misallocating tasks imposes massive opportunity costs. Conversely, organizations that master optimal agent allocation unlock exponential productivity gains.</p><p>At scale, with thousands of tasks and subtasks, manual delegation breaks down. Automation becomes essential. If an agent can reliably complete a task with high accuracy — and user permissions allow — the agent should be auto-selected and assigned.</p><p><strong>Automatic, correct delegation</strong> becomes the cornerstone of reliable automation.</p><figure class="w-richtext-align-center w-richtext-figure-type-image"><a href="https://www.youtube.com/watch?v=YhFj2zujcf8" target="_blank" class="w-inline-block"><div><img alt="Interloom Agent Delegation Demo" src="https://cdn.prod.website-files.com/68a579c332d0c68279830d51/68a5b1f54a2e08065be3ced6_Interloom-Agent-Assignment-Thumbnail.png" loading="lazy"/></div></a></figure><h3>Scaling Self-Service Agent Creation</h3><p>A few months ago, we introduced <strong>self-service agent creation</strong> via natural language instructions on our platform.</p><p>As more and more users were onboarded, the agent count grew rapidly, and quickly the fundamental delegation problem (<strong>Which agent is most suitable for a particular task?</strong>) became increasingly hard to solve automatically.</p><p>In a perfect world, this problem could have been modeled and solved as a supervised learning optimization — classifying tasks by agents.</p><p>In reality, conditions were far more complex due to:</p><ul role="list"><li>A wide and growing range of tasks (T)</li><li>A rapidly expanding and potentially unlimited set of user-created agents (A)</li><li>Tasks spanning diverse customer domains (D)</li></ul><p>For every task (T), the right agent (A) had to be selected across all domains (D). The complexity of the problem can be expressed as T × A × D — clearly an exponentially hard challenge.</p><p>Meanwhile, business needs demanded a solution that was <strong>flexible, reliable, and real-time</strong> across diverse domains.</p><h3>Evaluation &amp; Solution Approach</h3><p>Before testing solutions, we first focused on defining how to evaluate them.</p><p>Our evaluation aimed to answer:</p><ul role="list"><li>What is the accuracy of agent selection (Top-1 and Top-3)?</li><li>How does accuracy behave as the number of agents scales?</li></ul><p>Following <strong>Test-Driven Development</strong> principles, we created evaluation datasets.</p><p>Initially, we used synthetic datasets across five core domains:</p><ul role="list"><li>Customer support</li><li>Marketing outreach</li><li>Technical documentation</li><li>Sales enablement</li><li>Internal communications</li></ul><p>Later, we validated the approach with real customer datasets.</p><h2>Synthetic Dataset Creation</h2><p>Each agent was characterised by:</p><ul role="list"><li>Name</li><li>Description</li><li>Instructions</li><li>Tools</li></ul><p>To start, we focused on <strong>name</strong>, <strong>description</strong>, and <strong>sample user requests</strong>, creating a synthetic dataset consisting of 70 agents, each assigned five specific tasks. For example:</p><figure class="w-richtext-align-center w-richtext-figure-type-image"><div><img alt="Synthetic Dataset" src="https://cdn.prod.website-files.com/68a579c332d0c68279830d51/68a5b22760a6078d3b000be3_synthetic-dataset.png" loading="lazy"/></div></figure><p>Ground truth datasets were subsequently generated for evaluation purposes:</p><figure class="w-richtext-align-center w-richtext-figure-type-image"><div><img alt="ground-truth-dataset" For comparison random chance classification is only ~1%.</p><p>Despite the open-ended nature of the classification problem, we achieved strong baseline results. For comparison: random chance classification would yield only ~1% accuracy.</p><p><strong>Learning #2:</strong> Including <strong>user request examples</strong> in the delegation context <strong>significantly boosts performance</strong>.</p><p>To further improve results, we identified several potential next steps:</p><ul role="list"><li>Query expansion</li><li>Query decomposition</li><li>Model fine-tuning</li><li>Adoption of encoder-based architectures (e.g., ModernBERT)</li></ul><figure class="w-richtext-align-center w-richtext-figure-type-image"><div><img confirmed the strong potential of LLM-based solutions for agent delegation.</p><p>Key practical techniques included:</p><ul role="list"><li>Reranking candidate agents</li><li>Including user request examples</li><li>Leveraging model probability outputs for evaluation and decision-making</li></ul><p>While this solution already significantly improves the user experience on the <strong>Interloom platform</strong>, its impact reaches beyond that.</p><p>In <strong>multi-agent systems</strong>, an orchestration agent must reliably allocate and coordinate tasks across multiple agents.</p><p><strong>Reliability at this stage is critical</strong> — it directly determines the overall quality and efficiency of the system and is key to operationalize the AI agent advantages.</p><p>We have successfully deployed a similar solution in production, proving it to be:</p><ul role="list"><li><strong>Highly scalable</strong></li><li><strong>Adaptable across diverse domains</strong></li><li><strong>Robust and effective</strong> as a foundation for future innovations.</li></ul>
